engine¶

Pythonic wrapper around Lucene search engine.

Provides high-level interfaces to indexes and documents, abstracting away java lucene primitives.

indexers¶

Wrappers for lucene Index{Read,Search,Writ}ers.

The final Indexer classes exposes a high-level Searcher and Writer.

Indexer¶

class engine.indexers.Indexer(*args, **kwargs)¶

Bases: engine.indexers.IndexWriter

An all-purpose interface to an index. Opening in read mode returns an IndexSearcher. Opening in write mode (the default) returns an IndexWriter with a delegated IndexSearcher.

commit()¶
Commit writes and refresh searcher. Not thread-safe.

PrefixField¶

class engine.documents.PrefixField(name, store=False, index=True, termvector=False, sep='', start=1, stop=None, step=1)¶

Bases: engine.documents.NestedField

Field indexed with a prefix tree. Unlike a normal nested field, the field names only refer to the depth, while the values are nested. The component field names are expressed as slices of the original values, but may be customized.

Parameters:
  • start, stop, step – slice parameters, which respect the python convention of half-open intervals.
getname(index)¶
Return customized name for prefix field of given depth.
items(*values)¶
Generate tiered indexed fields along with the original value. Optimized to handle duplicate values.

Return lucene TermQuery of the appropriate prefixed field.
